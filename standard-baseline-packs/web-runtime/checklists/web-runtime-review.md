# Web Runtime Review Checklist

- [ ] Selected profile includes `web-app` or a Web-based `internal-admin`.
- [ ] Framework, router, and component boundaries are documented.
- [ ] State ownership is documented.
- [ ] API client boundary is documented.
- [ ] Loading, empty, error, and success states are covered.
- [ ] Build, typecheck, lint, test, and preview commands are known.
- [ ] Changed UI can be verified in a browser.
- [ ] Pack selection is not treated as implementation approval.
