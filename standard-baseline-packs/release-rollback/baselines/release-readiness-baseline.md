# Release Readiness Baseline

## Scope

Use this baseline when a project claims release readiness, handoff readiness, or staging/production deployment readiness.

## Required Decisions

- Release owner.
- Target environment.
- Release scope.
- Verification commands.
- Smoke checks.
- Evidence path.
- Stop conditions.

## Minimum Expectations

- Release scope is explicit.
- Verification evidence is recorded.
- Known limitations are listed.
- Human release decision is separate from AI recommendation.
- Rollback option is documented before release handoff.

## Boundary

This baseline does not approve release or production.
