# Internal Admin UI Baseline

## Scope

Use this baseline for admin consoles, management dashboards, approval flows, back-office tools, CRM, ERP, finance consoles, HR consoles, or internal operations systems.

## Expectations

- Favor dense, scannable, operational layouts over marketing layouts.
- Keep list, detail, form, approval, table, filter, search, pagination, and empty-state behavior explicit.
- Record role, permission, export, audit, destructive action, and production-data assumptions as pending unless confirmed.
- Route financial correctness, HR policy, tax, legal, and compliance decisions to humans.

## Boundary

This baseline does not approve permissions, finance, HR, tax, legal, compliance, sensitive export, production release, or target-project writes.
