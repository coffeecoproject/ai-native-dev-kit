# Mini Program Runtime Baseline

## Scope

Use this baseline when WeChat Mini Program pages, components, lifecycle, platform APIs, or runtime behavior are in scope.

## Expectations

- Keep page, component, and shared utility boundaries visible.
- Record login, permission, cloud, and platform API assumptions as pending until confirmed.
- Treat backend, cloud functions, payment, and release submission as conditional scope, not default scope.
- Verify core pages in the Mini Program toolchain or an equivalent documented runtime check when behavior changes.

## Boundary

This baseline does not approve backend work, payment, privacy, cloud access expansion, release submission, or target-project writes.
