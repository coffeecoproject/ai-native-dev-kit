# Backend API Evidence Policy

Evidence should point to project files, commands, migration reports, API tests, or integration smoke results from the target project.

Pack files are not project evidence.

## Required For Confirmed Selection

- Selected profile or backend capability evidence.
- BL level evidence.
- API contract evidence.
- Domain type and schema decision evidence.
- Local or safe-environment verification evidence.

If evidence is unavailable, record `PENDING` instead of inferring readiness.
