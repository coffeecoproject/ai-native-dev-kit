# Backend API Review Checklist

- [ ] Selected profile includes `backend-api` or backend capability is documented.
- [ ] API contract boundary is documented.
- [ ] DTO/domain boundary is documented.
- [ ] Error model is documented.
- [ ] Auth integration point is documented.
- [ ] Migration and rollback expectations are documented where data changes exist.
- [ ] Local or safe-environment verification command is known.
- [ ] Pack selection is not treated as implementation approval.
