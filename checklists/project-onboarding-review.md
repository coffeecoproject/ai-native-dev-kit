# Project Onboarding Review Checklist

Use this checklist before starting the first non-trivial implementation task.

## Documents

- [ ] `docs/project-onboarding.md`
- [ ] `docs/project-profile.md`
- [ ] `docs/tech-stack-strategy.md`
- [ ] `docs/business-spec-index.md`
- [ ] `docs/sample-policy.md`
- [ ] `docs/onboarding-decisions.md`

## Human Decision Role

- [ ] AI drafted the documents from conversation
- [ ] Human was asked focused decision questions, not asked to manually fill all files
- [ ] Human confirmed or rejected key assumptions
- [ ] Open decisions are recorded
- [ ] Risk acceptances are explicit

## Project Context

- [ ] Project type is clear
- [ ] Target platform or platforms are clear
- [ ] Primary users or actors are clear enough
- [ ] First valuable outcome is clear
- [ ] First vertical slice candidate is identified

## Technology Strategy

- [ ] Selected stack is approved or intentionally deferred
- [ ] Dependency policy is stated
- [ ] Verification commands or categories are stated
- [ ] Deployment or distribution assumptions are marked pending if unknown

## Business Scope

- [ ] Capability map exists
- [ ] Domain entities are identified when relevant
- [ ] Access or permission questions are identified
- [ ] High-risk business rules are identified
- [ ] Deferred scope is explicit

## Sample Policy

- [ ] Samples are not created before project context is clear enough
- [ ] Samples use fake data
- [ ] Samples are marked non-production
- [ ] Samples are not promoted into shared dev-kit examples

## Decision

Choose one:

- [ ] `READY_FOR_FIRST_REQUEST`
- [ ] `NEEDS_MORE_COMMUNICATION`
- [ ] `BLOCKED_BY_DECISION`
- [ ] `BLOCKED_BY_RISK`
