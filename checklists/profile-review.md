# Profile Review Checklist

Use this checklist before adding or changing `profiles/<name>/profile.md`.

## Structure

- [ ] Has `Purpose`
- [ ] Has `Applies To`
- [ ] Has `Does Not Apply To`
- [ ] Has `Default Task Level`
- [ ] Has `Required Project Docs`
- [ ] Has `Focus Areas`
- [ ] Has `Platform / Project-type Risks`
- [ ] Has `High-risk Boundaries`
- [ ] Has `Required Verification`
- [ ] Has `Release / Distribution Checks`
- [ ] Has `AI Boundaries`
- [ ] Has `Starter Expectations`

## Layering

- [ ] Does not include concrete project facts
- [ ] Does not duplicate core workflow rules unnecessarily
- [ ] Does not weaken core gates
- [ ] Does not place platform/tool instructions in the wrong layer

## Verification

- [ ] Has concrete verification categories
- [ ] Has stop conditions for missing verification
- [ ] Identifies high-risk areas that require human approval

## Decision

APPROVE / REQUEST_CHANGES / BLOCK
