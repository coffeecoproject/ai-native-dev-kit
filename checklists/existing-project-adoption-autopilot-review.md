# Existing Project Adoption Autopilot Review

Use this checklist when reviewing an adoption autopilot report.

- The report is read-only.
- User-facing sections are understandable without internal IntentOS subsystem
  names.
- The report does not claim full adoption.
- The report does not claim project authority changed.
- The report does not claim native assets were installed.
- The report does not claim target files were written.
- Any next step is framed as a plain permission/risk decision.
- Technical trace is present for audit but does not replace the human summary.
- Machine-readable evidence validates against the schema.
