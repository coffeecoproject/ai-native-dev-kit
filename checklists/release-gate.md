# Release Gate Checklist

- [ ] Release owner assigned
- [ ] Related specs/tasks linked
- [ ] Verification evidence attached
- [ ] Migration reviewed
- [ ] Rollback plan documented
- [ ] Feature flag reviewed
- [ ] Logs/metrics/alerts reviewed
- [ ] Project automations reviewed
- [ ] Automation proposals approved or explicitly deferred
- [ ] Automation status reviewed: ACTIVE / PAUSED / not needed
- [ ] Known issues documented
- [ ] Post-release checks defined
- [ ] Approval recorded for L2/L3 changes

Do not release L3 changes without explicit approval and rollback evidence.
Do not create, update, resume, delete, or enable recurring automations during release without explicit approval.
