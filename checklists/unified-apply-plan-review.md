# Unified Apply Plan Review Checklist

- [ ] The plan starts with a clear plain outcome summary and user-input class.
- [ ] The plan says Codex cannot write now.
- [ ] Every planned action has target paths, reason, status, approval requirement, and rollback requirement.
- [ ] `Will write now` is `No` for every action.
- [ ] High-risk actions are specialized, exact-authority-bound, or blocked.
- [ ] Source evidence is linked or explicitly marked missing / not applicable.
- [ ] Backup, rollback, and verification are listed before any apply step.
- [ ] Existing governed project rules are mapped instead of replaced.
- [ ] The plan does not approve implementation, release, production, hooks, CI, secrets, migrations, industrial packs, legal, security, privacy, or compliance decisions.
- [ ] The outcome is one of `APPLY_PLAN_RECORDED`, `NEEDS_HUMAN_DECISION`, or `BLOCKED`.
