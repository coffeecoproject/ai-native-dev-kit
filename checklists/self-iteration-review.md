# Self-iteration Review Checklist

Use this checklist before accepting workflow improvements or intentos change proposals.

## Evidence

- [ ] Linked AI task logs exist
- [ ] Linked workflow retro exists when applicable
- [ ] Problem is repeated or high-impact
- [ ] Impact is described
- [ ] Proposed layer is justified

## Layering

- [ ] Project-specific issue stays in project docs
- [ ] Platform-specific issue stays in profile/starter
- [ ] Generic workflow issue may enter templates/checklists/core
- [ ] Core is used only when narrower layers are insufficient

## Safety

- [ ] Does not weaken high-risk gates
- [ ] Does not remove human approval
- [ ] Does not reduce verification evidence
- [ ] Does not introduce business assumptions into generic assets

## Decision

APPROVE / REQUEST_CHANGES / REJECT / DEFER
