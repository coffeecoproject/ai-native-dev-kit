# Platform Release Recipe Review

- [ ] Recipe status is `STRICT` or `DRAFT`.
- [ ] Strict recipes include release owner, rollback, monitoring, environment, and evidence requirements.
- [ ] Draft recipes are clearly marked and do not pass strict checks.
- [ ] Codex allowed actions are local-safe only.
- [ ] Provider APIs, uploads, preview publication, remote mutation, production deploy, app-store submission, mini-program review/release, migrations, DNS, payment, permissions, secrets, and production config remain blocked.
- [ ] Recipe does not approve release or make Codex the release owner.
- [ ] Recipe explains confidence and platform signals without turning assumptions into certainty.
- [ ] Release Execution bridge is plan-first.
