# Control Effectiveness Review

- [ ] The bounded claim is explicit and does not expand from a control name.
- [ ] Implementation, configuration, and strict consumer refs resolve safely.
- [ ] Every ref and result is bound to the current project, task, and revision.
- [ ] Observed scope matches a current inventory and exclusions have evidence.
- [ ] The implemented assertion semantically matches the declared claim.
- [ ] Strict blocking claims have safe representative failure proof.
- [ ] Dynamic proof uses a bounded direct-process adapter and proves cleanup.
- [ ] Partial proof remains bounded and is not consumed as full proof.
- [ ] Project ownership and IntentOS origin do not imply effectiveness.
- [ ] The report does not authorize writes, release, production, or completion.
- [ ] The ordinary user is not asked to make a technical control decision.
