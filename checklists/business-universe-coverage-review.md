# Business Universe Coverage Review

- [ ] Work Queue, Task Governance, task, intent, refs, and digests match exactly.
- [ ] Conditional routing uses an evidenced generic reason code.
- [ ] A bounded non-behavioral task is not burdened with an empty report.
- [ ] Partial, unsupported, or truncated discovery remains blocked and resumable.
- [ ] Every relevant category, participant, origin, and path has evidence and one disposition.
- [ ] Lifecycle applicability includes failure, recovery, correction, reversal, termination, and observation when relevant.
- [ ] A not-applicable lifecycle stage has a project-local reason and evidence.
- [ ] Selection points and consistency groups are evidence-bound when applicable.
- [ ] Path provenance distinguishes runtime, automation, external, fixture/seed, mock/stub, and manual-only paths.
- [ ] Mock, seed, fixture, or manual evidence is not used as runtime proof.
- [ ] Every scenario ID and digest are derived from canonical scenario semantics.
- [ ] Every scenario states expected and negative or reverse behavior.
- [ ] Required proof strength matches the path and lifecycle behavior.
- [ ] Evidence locators resolve safely inside the project and match Evidence Authority.
- [ ] Technical uncertainty remains Codex-owned; business and external facts are classified correctly.
- [ ] Required Challenger evidence binds the current task, intent, discovery
  boundary, and exact scenario-id/digest set, with no unresolved finding.
- [ ] Every downstream consumer preserves the exact scenario set and source IDs.
- [ ] Positive and negative or reverse verification obligations exist for each scenario.
- [ ] A partial discovery records inventory, cursor, candidate-set, and resume-state digests; a resumed run preserves the prior report and converges deterministically.
- [ ] Platform verification is derived from project paths/profiles and does not force client-only behavior into backend/API proof.
- [ ] The report does not authorize implementation, completion, release, or production.
