# Claim Control Review Checklist

Use this checklist before publishing README, release records, handoffs, launch notes, or public summaries.

## Required Sections

- Allowed Claims
- Forbidden Claims
- Evidence Status
- Known Limitations
- Verification

## Overclaim Scan

Reject wording that says or implies:

- guaranteed safe
- production proven without production evidence
- suitable for every project
- no human approval required
- report approves release
- simulated dogfood proves production readiness
- draft pack is stable
- all industrial packs are enabled by default

## Evidence Fit

- If evidence is simulated, say simulated.
- If evidence is generated-project smoke, say generated-project smoke.
- If evidence is draft, say draft.
- If evidence is advisory, say advisory.
- If human decision is pending, say pending.

## Human Boundary

- Claims about release, production, legal, customer delivery, operations, or risk acceptance need human confirmation.
