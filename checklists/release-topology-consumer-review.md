# Release Topology Consumer Review

- [ ] Every strict release consumer references the same topology report.
- [ ] The topology file digest and canonical digest both match.
- [ ] Project identity and source revision match the current project.
- [ ] Release Evidence, Approval, Execution, and Runtime bindings agree.
- [ ] Legacy Release Channel evidence is compatibility-only.
- [ ] Two active publishing triggers are blocked or explicitly reconciled.
- [ ] Candidate and package identities remain consistent across consumers.
- [ ] Embedded consent is not treated as release authority.
- [ ] Generated projects enforce the same strict semantics.
- [ ] The user was not asked to make a technical topology choice.
