# Request 140: Project Memory & Context Governance

## Human Summary

Add a formal project memory and context governance layer so project facts become more accurate over time without letting Codex self-learn or persist unconfirmed assumptions.

## Request

Create a complete 1.4.0 phase that formalizes:

- Git Boundary
- Context Governance
- Learning Candidate
- Context Correction Report
- Git Boundary Report
- deterministic context governance checks
- examples, bad fixtures, and release evidence

## Non-goals

- Do not build Safe Launch / Delivery Readiness in this phase.
- Do not add external memory, vector database, or API automation.
- Do not store raw conversations.
- Do not let Codex write unconfirmed candidates into baselines.
- Do not claim automated secret scanning beyond deterministic obvious-pattern checks.

## Human Decision

Status: Approved for implementation.

Scope: 1.4.0 project memory and context governance only.
