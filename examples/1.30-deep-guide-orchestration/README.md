# 1.30 Deep Guide Orchestration Example

This example shows the 1.30 `guide --deep` behavior.

The user still receives one Workflow Guidance Card. Internally, Codex may select read-only resolver results such as review surface, delivery path, work queue, document lifecycle, workflow mapping, baseline decision, or hook policy.

The example proves that deep guide orchestration stays read-only and does not ask the user to choose internal workflow commands.
