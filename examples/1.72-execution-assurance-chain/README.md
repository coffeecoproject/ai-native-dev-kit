# 1.72 Execution Assurance Chain Examples

These examples show how IntentOS verifies execution-class work from an evidence chain.

- `feature-contract-validation`: cross-surface feature execution, `VERIFIED_DONE`.
- `old-project-intentos-adoption`: old-project adoption execution, `VERIFIED_DONE`.
- `safe-copy-patch`: narrow low-risk patch, `VERIFIED_DONE`.
- `patch-smell-backend-only`: backend-only validation patch smell, blocked.

The examples are synthetic and business-neutral.
