# Controlled Native Adoption Review Examples

These examples exercise the read-only maturity and adoption-depth review with
the current 1.109 project-entry and authority rules. Directory names are kept
for historical fixture compatibility; the generated reports carry the current
recommendation semantics.

They are intentionally small project shapes:

- `strong-governed-stay-partial`: has strong governance signals, but missing
  current project-bound authority evidence routes to authority reconciliation;
  strong governance no longer forces permanent partial adoption.
- `weak-governance-repair`: has partial governance signals and must reconcile
  current authority before dependent repair and overlay actions.
- `messy-production-repair-only`: looks production-sensitive and incomplete,
  so a bounded governance-repair and selected-overlay plan comes first.
- `light-plan-only`: is low risk enough for a selected behavior-complete
  overlay plan, while the review itself still cannot apply files.

The reports are generated by `scripts/resolve-controlled-native-adoption-review.mjs`
and checked by `scripts/check-controlled-native-adoption-review.mjs`.
