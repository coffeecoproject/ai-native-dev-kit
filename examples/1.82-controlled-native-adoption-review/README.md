# Controlled Native Adoption Review Examples

These examples exercise the 1.82 read-only maturity and adoption-depth review.

They are intentionally small project shapes:

- `strong-governed-stay-partial`: already has strong governance, so IntentOS should stay as a planning/review overlay.
- `weak-governance-repair`: has some governance but not enough to deepen adoption safely.
- `messy-production-repair-only`: looks production-sensitive and incomplete, so repair comes first.
- `light-plan-only`: low-risk project where a selected deeper adoption plan may be prepared, but not applied.

The reports are generated by `scripts/resolve-controlled-native-adoption-review.mjs`
and checked by `scripts/check-controlled-native-adoption-review.mjs`.
