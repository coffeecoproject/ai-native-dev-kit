# Plan Review Gate Examples

These examples show how IntentOS treats implementation plans as reviewable
artifacts before coding starts.

- `low-docs-plan-skip`: low-risk docs work with Task Governance-backed skip.
- `medium-ui-plan-reviewed`: bounded UI/API plan that passes targeted review.
- `high-permission-delete-plan-revision`: high-impact permission/data deletion
  plan that needs revision.
- `high-permission-delete-plan-passed`: revised high-impact plan that passes
  the pre-implementation plan review prerequisite.
- `high-business-rule-plan-stale`: stale plan identity blocks implementation
  review.

All examples remain non-authorizing: they do not approve implementation,
commit, push, release, production, tests, migrations, or project writes.
