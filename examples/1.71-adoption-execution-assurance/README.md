# 1.71 Adoption Execution Assurance Examples

These examples show how existing-project adoption claims are checked from evidence.

- `verified-existing-project`: all required surfaces are mapped or project-owned, and simulation passed.
- `partial-existing-project`: IntentOS can operate plan-first, but simulation or work queue evidence is missing.
- `blocked-production-project`: project authority blocks full adoption until release / CI ownership is resolved.
- `failed-assurance`: intentionally unsafe example for review calibration.
