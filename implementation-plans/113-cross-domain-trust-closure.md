## Scope

- Enforce the unified task consumer chain for changed task and release surfaces.
- Bind evidence to the exact project, task, candidate content, implementation, and source chain.
- Make controlled apply transactional, failure-atomic, recoverable, and exact-action bounded.

- Keep release planning and evidence bounded; do not grant deploy, publish, production, secret, DNS, payment, migration, or external-system authority.
- Generate an exact source-only candidate, runtime-hygiene record, channel policy, release-evidence view, and `PLAN_ONLY` execution record so source review is evidenced without being mistaken for an external release.
- Preserve task-scoped verification outputs generated by the bounded runtime run, including the Web BL2 project-native proof log under `examples/web-industrial-bl2-first-slice/.intentos/runtime-runs/`, as evidence-only artifacts with no external effect.
- Preserve zero-experience solo-user behavior: Codex decides technical architecture, baseline, verification, and review routing.
- Backend authority and capability source remain the project-owned Node CLI and checker implementations; generated guidance and UI-facing summaries are derived consumers and cannot override those source implementations.
