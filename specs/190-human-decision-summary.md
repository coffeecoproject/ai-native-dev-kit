# Spec 190: Human Decision Summary

## Status

Done

## Source

- Request: `requests/190-human-decision-summary.md`
- Preflight: `preflight/190-human-decision-summary.md`

## Problem

AI Native Dev Kit can correctly classify adoption and governance paths, but the human-facing output may still feel like a technical recommendation instead of a decision brief.

## User Story

As a non-technical or low-context project owner,
I want Codex to present adoption, baseline, migration, review, and launch recommendations as clear options,
so I can decide without first learning internal workflow status codes.

## Scope

Included:

- Output Experience Protocol upgrade.
- Reporter and related agent prompt updates.
- Decision/status/review/adoption/baseline/template updates.
- `workflow-next`, `start`, `baseline`, and governance migration report output updates.
- Docs and README usage expectation updates.
- Output quality and dev-kit checker calibration.
- 1.9.0 version and release evidence.

## Non-goals

- No new approval authority.
- No automatic GPT/API review hook.
- No target-project auto-scanner.
- No change to read-only governed-project protection.
- No change to baseline apply boundary.
- No private project naming in public evidence.

## Data Model Impact

No runtime data model change.

Template/report sections add:

- `## Human Decision Summary`
- recommended choice
- options table
- file-write impact
- no-decision outcome

## API / Interface Contract

Human output changes:

- `workflow-next --format human` prints `Human Decision Summary` before technical fields.
- `start-project.mjs` prints `Human Decision Summary` before classification details.
- `baseline-project.mjs` prints `Human Decision Summary` before baseline details.
- migration reports generated by `init-project.mjs` include `Human Decision Summary`.

Machine-readable JSON output should remain compatible.

## UI States

Not applicable.

## Permission Rules

- Reports are not approvals.
- Human Decision Summary cannot authorize release, risk acceptance, production config, migration, dependency, payment, privacy, security, or AGENTS/PR template migration by itself.
- Any option that writes files must say what kind of files it writes.

## Observability

- `scripts/check-dev-kit.mjs` checks protocol and script markers.
- `scripts/score-output-quality.mjs` scores report outputs for decision clarity.
- Release evidence records known limitations.

## Acceptance Criteria

- Output protocol requires `Human Decision Summary`.
- High-frequency templates include the decision summary section.
- Reporter and workflow prompts instruct agents to start decision-heavy output with the summary.
- `workflow-next`, `start`, `baseline`, and migration reports show recommended option tables.
- Docs explain that users choose options and do not need to read internal status codes first.
- Full dev-kit checks pass.
