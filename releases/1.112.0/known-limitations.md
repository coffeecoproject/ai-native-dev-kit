# IntentOS 1.112.0 Known Limitations

- Behavioral closure is not accepted. See `domain-audit-report.md`.
- Public finish can currently select a completion-reporting action before a
  failed mandatory source dominates.
- Optional Business Universe routing can remove minimum non-HIGH business,
  impact, verification, and completion obligations.
- Applicable Change Boundary and Standard Baseline Selection consumers can
  currently pass without reports on compatibility routes.
- Completion Evidence does not independently execute strict Execution
  Assurance validation.
- The current effective guidance graph omits some runtime outputs that can
  still delegate technical judgment to generic human decisions.
- Public release-chain continuity, future-session adoption activation, and
  process-interruption apply recovery remain unproved end to end.
- Repository and distribution structure remains intentionally unchanged until
  the P0/P1 closure work is accepted.
- The audit cannot prove absolute business or production correctness, and it
  does not authorize any external effect.
