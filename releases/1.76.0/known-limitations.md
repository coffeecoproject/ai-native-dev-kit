# IntentOS 1.76.0 Known Limitations

- Verification Plans are derived from Business Rule Closure, Change Impact Coverage, project level, platform hints, and task intent. They do not prove that the generated tests are perfect.
- The checker validates plan structure, source binding, obligations, and overclaim boundaries. It does not execute test suites.
- Strict mode is explicit. Older Markdown records remain compatible unless maintainers require structured evidence.
- Broad command rejection is conservative. A broad command can still be listed as supporting evidence, but not as the only proof for required obligations.
- Manual verification ownership is checked structurally; IntentOS does not verify real human identity.
- 1.76.0 does not replace Execution Assurance. Completion claims still need the execution assurance chain when Codex says execution-class work is done.
