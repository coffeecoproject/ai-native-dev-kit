# 1.67.0 Known Limitations

- Release Plan is a view layer. It does not prove that a release is safe.
- Release Plan does not replace source-system records.
- Existing project rule comparison is only as strong as the project evidence Codex can read.
- Full old-project asset migration still requires Native Migration, Existing Rule Reconciliation, Unified Apply Plan, approval, and Controlled Apply Readiness.
- Release Plan intentionally does not execute provider commands, deploy, publish, submit app/mini-program review, mutate remote state, or manage secrets.
