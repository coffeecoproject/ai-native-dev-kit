# IntentOS 1.109.0 Known Limitations

- Complete project reading is bounded by observable repository and runtime
  evidence. Undocumented external rules or systems cannot be inferred with
  certainty.
- Same-run evidence is intentionally ephemeral. Durable adoption authority
  still requires the corresponding project-bound artifact and controlled apply
  evidence when project files will change.
- Behavior-complete adoption does not mean every IntentOS asset replaces every
  project-native asset. Stronger compatible project rules remain authoritative.
- Atomic bootstrap protects IntentOS-owned setup actions. It cannot roll back
  unrelated concurrent external changes and will stop when target topology
  changes after preflight.
- Generated-project cold start proves the installed operating route is usable;
  it does not prove the generated product, business universe, tests, release
  path, or production environment is complete.
- A zero-experience user can still be asked for a genuinely unavailable
  business fact, an authoritative external fact, or exact consent to a prepared
  real-world effect. Technical uncertainty remains Codex-owned work.
- This release inventories and binds relied-on controls but does not prove that
  every declared control is behaviorally effective. That work remains outside
  the 1.109 claim boundary.
