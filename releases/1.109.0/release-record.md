# IntentOS 1.109.0 Release Record

## Theme

Project Entry and Behavior-Complete Adoption Trust Hardcut.

## Delivered

- one path-safe target topology and project identity used by public entry,
  baseline, adoption, and setup consumers;
- one project-fact projection for lifecycle evidence, selected platforms,
  baseline posture, effective Guidance Authority, governance inventory, and
  current-work continuity;
- exact preservation of the original natural-language goal and current task
  across setup, adoption, interruption, and restart;
- same-run, project-bound, digest-bound evidence transport from Native
  Migration through Rule Reconciliation, Governance Convergence, controlled
  adoption review, and Adoption Assurance;
- strict existing-project adoption that preserves stronger project authority,
  scopes dirty-work conflicts to overlapping writes, and keeps technical
  migration decisions inside Codex;
- atomic absent-target and empty-directory setup with topology recheck, exact
  action graph, rollback, interrupted-run recovery, and a project-local
  tamper-evident bootstrap receipt;
- generated-project cold start and first ordinary task routing from project-
  local IntentOS assets before behavioral activation can be claimed;
- continuous project identity across exact approved workflow updates, with one
  bounded activation capability during the receipt-write window and a verified
  bootstrap-plus-update evidence chain after completion;
- one current-task projection when the same Work Queue item appears in both a
  summary table and its detailed item table;
- generic calibration fixtures for blank targets, light existing projects,
  governed projects, current-work conflicts, stale evidence, and generated
  project parity;
- strict source-chain regression tests, including copied/stale evidence,
  symlinked receipts, partial apply failure, orphan recovery, Business Universe
  preservation, new/legacy controlled-update parity, and zero-experience entry
  behavior.

## Allowed Claims

- Public entry consumers can agree on one current target, project identity,
  fact projection, goal, Guidance Authority view, and current-work state.
- A blank target can be initialized through an atomic, recoverable transaction
  and rejected without residual files when setup or activation fails.
- Existing-project adoption can reuse one strict same-run evidence chain without
  asking the user to choose a technical adoption mode.
- File installation alone cannot count as verified behavioral activation.

## Forbidden Claims

- Project entry or adoption proves product, business, test, or production
  correctness.
- Behavior-complete adoption requires replacing stronger project-owned rules.
- A bootstrap receipt authorizes release, provider actions, production, cost,
  secrets, migrations, or irreversible data effects.
- Calibration fixtures prove every repository shape or governance control.
- This release proves that a declared project-native control is effective;
  bounded control effectiveness remains a later concern.

## Boundaries

This release changes project-entry, setup, adoption, and generated-project
trust contracts. It does not deploy software, change production, replace
project-owned release authority, or approve a real-world effect.

## Evidence Status

The release counts as complete only when the exact 1.109 candidate passes the
project-entry suite, strict Phase 0 evidence chain, generated-project parity,
Manifest and Review Context checks, full repository verification, independent
review, and diff/boundary checks.
