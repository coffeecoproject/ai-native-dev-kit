# 1.18.1 Known Limitations

- Platform States are now required and checked, but they are still selector
  output and require human confirmation for real project adoption.
- The precision scoreboard is a calibration aid, not production validation.
- Existing sanitized calibration reports are local and read-only.
- BL2 remains candidate-only until selected packs, evidence, residual risk, and
  human approval are recorded.
- Remote CI run evidence is not embedded until a GitHub Actions run URL is
  recorded after push.
- No target-project write approval, implementation approval, release approval,
  production approval, or real-project production validation is included.
