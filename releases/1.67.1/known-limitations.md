# 1.67.1 Known Limitations

- Release Plan remains a pure view model. It does not approve release, execute provider commands, mutate CI/hooks, or write target-project files.
- Existing-project IntentOS Operating Mode does not grant migration permission. Governance asset changes still require Native Migration, Existing Rule Reconciliation, Unified Apply Plan, Approval Record, and Controlled Apply Readiness.
- Rule comparison can recommend keeping, merging, or adopting rules, but it does not decide business, legal, production, release, finance, tax, payment, privacy, permission, data, or compliance authority.
- Strict `release_plan_digest` validation applies to structured Release Plan evidence. Historical Markdown-only records remain compatible unless strict evidence mode is requested.
- WorkControl-style production projects can be read and routed through IntentOS mode, but full native takeover remains a reviewed migration decision, not an automatic bootstrap.
