# Release Approval Record

A Release Approval Record is the only human-owned release authorization input
accepted by IntentOS Release Execution.

It binds one approval to the current project, Git revision, release target,
candidate file and digest, package identity, strict release evidence, runtime
hygiene, channel policy, platform recipe or handoff when required, release
owner, SOP, rollback, monitoring, and post-release smoke evidence.

Ordinary prose, chat text, tags, GitHub Releases, Launch Review View, Release
Plan, recipe status, and command-line approval flags are not release approval.

The record may authorize only bounded low-risk assistance. Production deploy,
store or mini-program submission, migration, secrets, DNS, payment,
permissions, production config, and rollback execution remain human or
external-system owned.
