# Codex Mini Program Industrial Execution

## Before Implementation

- read `AGENTS.md`
- read project baseline selection and evidence
- resolve selected profiles and industrial packs
- identify platform permission, cloud function, storage, API, release, and value-transfer risk
- stop for human approval when the task touches release submission, production config, value transfer, or platform permissions

## During Implementation

- keep the task scope narrow
- preserve project-local structure and platform conventions
- update verification notes for critical flows, denied states, and cloud boundaries
- record exceptions rather than silently weakening the baseline

## Before Final Report

- run available verification
- include runtime, permission, cloud, or documented non-applicability evidence
- list release, privacy, and residual-risk impacts
