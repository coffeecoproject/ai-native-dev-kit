# Mini Program Runtime Quality Audit

## Audit Areas

- Page lifecycle behavior for changed pages.
- Loading, empty, success, error, forbidden, denied-permission, and recovery states.
- Navigation, return paths, tab switching, share entry, scan entry, and re-entry behavior when touched.
- Weak network, timeout, duplicate tap, retry, and offline-adjacent behavior.
- Platform capability usage and non-applicability reasons.
- Evidence quality for developer tool, simulator, real-device, test, or manual review evidence.

## Output

Record:

- confirmed runtime assumptions
- flows covered by evidence
- flows missing evidence
- platform capability risks
- first controlled follow-up slice
