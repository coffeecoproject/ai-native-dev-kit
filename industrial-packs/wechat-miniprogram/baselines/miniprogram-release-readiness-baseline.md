# Mini Program Release Readiness Baseline

## Required Coverage

- Experience version, preview, or review-like validation exists before release submission when release is in scope.
- Platform category, permission declaration, privacy declaration, and review notes are checked when changed.
- Release scope, rollback or mitigation path, monitoring path, and incident owner are documented.
- Known platform limitations, review risk, and residual risks are accepted by a human.
- High-risk release, production configuration, value transfer, or privacy-impacting work is approved before implementation and release.

## Evidence Expectations

- Release evidence must identify the target version or release candidate.
- Evidence must include review readiness, rollback/mitigation, monitoring, and residual-risk notes.
- Release evidence must not be used to imply production approval when the record is example-only or review-only.

## AI Boundary

AI may prepare release records, review checklists, rollback notes, and incident drafts.

AI must not submit a mini program for review, approve release, approve category/privacy declarations, or accept residual release risk.
