# Mini Program Release Checklist

- [ ] Runtime assumptions and platform capability usage are documented.
- [ ] Critical flows have available runtime evidence.
- [ ] Permission prompts and denied states are covered.
- [ ] Cloud function, API, and storage boundaries are reviewed.
- [ ] Production configuration is reviewed by a human.
- [ ] Release submission and rollback evidence exists.
- [ ] Monitoring or failure observation evidence exists.
- [ ] Exceptions and residual risks are accepted by a human.
