# Mini Program Release Review Checklist

- [ ] Experience version, preview, or review-like validation exists when release is in scope.
- [ ] Platform category, permission declarations, privacy declarations, and review notes are checked when changed.
- [ ] Release scope is narrow and linked to approved task artifacts.
- [ ] Rollback or mitigation path is documented.
- [ ] Monitoring or incident observation path is documented.
- [ ] Human approval exists for release, production config, platform permission, value transfer, and residual risk when touched.
