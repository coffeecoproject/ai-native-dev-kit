# API Security And Release Checklist

- [ ] API contract impact is documented.
- [ ] Validation and error behavior are verified.
- [ ] Authentication and authorization are enforced server-side.
- [ ] Data writes and external side effects are covered by tests or evidence.
- [ ] Production configuration is reviewed by a human when changed.
- [ ] Rollback and observability evidence exists.
- [ ] Exceptions and residual risks are accepted by a human.
