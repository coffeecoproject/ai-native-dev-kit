# Web Security Checklist

- [ ] Protected behavior is not enforced only in the UI.
- [ ] Server-side permission checks are identified.
- [ ] Rejection paths are tested or manually verified.
- [ ] Resource scope boundaries are documented.
- [ ] Client-side code does not contain production secrets.
- [ ] Production config changes are reviewed.
- [ ] Cookie, session, CORS, CSRF, XSS, and security header impacts are reviewed when touched.
- [ ] Auth, permission, secrets, and production config changes have human approval.
