# Web Performance And Accessibility Checklist

- [ ] Critical desktop behavior is checked.
- [ ] Critical mobile behavior is checked.
- [ ] Primary controls can be reached by keyboard when the UI is interactive.
- [ ] Focus order and visible focus are reviewed for critical interactions.
- [ ] Interactive controls have understandable text, labels, or accessible names.
- [ ] Status, error, and validation messages are perceivable without relying only on color.
- [ ] Layout overlap, unstable layout, and blocked primary actions are reviewed.
- [ ] Bundle, asset, image, font, or dependency impact is reviewed when touched.
- [ ] Loading and interaction responsiveness are reviewed for critical flows.
- [ ] Skipped checks have rationale and residual risk owner.
