# Web UI State Checklist

- [ ] Critical routes or screens are listed.
- [ ] Loading states are covered.
- [ ] Empty states are covered.
- [ ] Error states are covered.
- [ ] Forbidden states are covered when protected resources exist.
- [ ] Success states are covered.
- [ ] Desktop behavior is checked for critical flows.
- [ ] Mobile behavior is checked for critical flows.
- [ ] API failure behavior is checked.
- [ ] Skipped checks have rationale and residual risk owner.
