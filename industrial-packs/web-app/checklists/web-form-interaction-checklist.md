# Web Form And Interaction Checklist

- [ ] Primary user actions have success behavior.
- [ ] Primary user actions have failure behavior.
- [ ] Forms have validation behavior for required, invalid, and boundary values.
- [ ] Submit buttons or commands prevent accidental duplicate submission when needed.
- [ ] Destructive or irreversible actions have confirmation and cancellation behavior.
- [ ] User-visible errors preserve enough context for recovery.
- [ ] Route changes, refresh, or back/forward navigation do not silently lose critical work.
- [ ] Skipped checks have rationale and residual risk owner.
