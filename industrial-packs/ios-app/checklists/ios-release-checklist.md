# iOS Release Checklist

- [ ] Supported OS and device matrix is documented.
- [ ] Critical flows have simulator or device evidence.
- [ ] Offline and degraded network states are covered where relevant.
- [ ] Privacy permissions and sensitive data handling are reviewed.
- [ ] Local persistence and cache behavior are verified where relevant.
- [ ] Signing and release configuration are reviewed by a human.
- [ ] Crash or failure monitoring evidence exists.
- [ ] Exceptions and residual risks are accepted by a human.
