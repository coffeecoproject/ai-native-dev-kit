# Preflight: Artifact Frontmatter + Schema

## Source Request

Request: `requests/039-artifact-frontmatter-schema.md`

## Clarity

READY

## Problem Summary

Artifact checks need machine-readable metadata, but existing human-readable Markdown artifacts must keep working during a time-boxed migration window.

## Missing Information

- Full migration of old examples is deferred to `0.40.x`.
- Strict schema as default is deferred until migration evidence exists.
- Checker library refactor is deferred to `0.40.1`.

## Assumptions

- A dependency-free frontmatter parser is sufficient for initial metadata.
- Schemas can be simple JSON contracts consumed by local validation code.
- New artifacts generated by `new-workflow-item.mjs` can include frontmatter without changing templates.
- Legacy artifacts should warn by default and fail only with `--strict-schema`.

## Direction Risks

- Schema requirements could make normal authoring too heavy.
- Frontmatter checks could conflict with existing Goal Mode, Review Loop, or artifact section semantics.
- Legacy compatibility could become permanent if no strict mode or migration warning exists.

## Over-design Risks

- Adding a full JSON Schema validator dependency is unnecessary for this phase.
- Migrating every example now would blend this phase with fixture expansion.
- Replacing Markdown sections with metadata would hurt human readability.

## MVP Recommendation

Add schemas, a lightweight frontmatter helper, generator metadata, checker validation, strict-schema mode, and self-check coverage for new, invalid, and legacy artifacts.

## Non-goals

- No full example migration.
- No default strict schema for all old artifacts.
- No dependency addition.
- No checker library refactor.
- No fixture matrix expansion.

## Domain Model Draft

- Artifact frontmatter: top-of-file metadata for type, number, slug, refs, status, and level.
- Artifact schema: JSON contract for required metadata fields.
- Legacy artifact: readable Markdown artifact without frontmatter.
- Strict schema mode: migration rehearsal mode that fails legacy artifacts.

## Permission / Security Risks

No secrets, auth, permissions, production configuration, data migration, destructive operation, value transfer, or dependency addition are touched.

## First Vertical Slice

Generate a request with frontmatter, verify checker accepts it, remove a required field to verify failure, then remove frontmatter to verify warning by default and failure with `--strict-schema`.
