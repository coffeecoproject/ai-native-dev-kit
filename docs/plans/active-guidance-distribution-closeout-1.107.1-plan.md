# IntentOS 1.107.1 Active Guidance Distribution Close-Out Plan

## Purpose

Close the gap between the zero-experience solo-developer contract and every
current instruction that Codex can read, reference, generate, or distribute.

This is a patch close-out. It does not add a new user entry, workflow state, or
execution authority.

## Product Boundary

- Codex is the only formally supported IntentOS agent platform in 1.107.1.
- Claude and Cursor assets remain compatibility references only.
- Compatibility references are not active guidance, default installed assets,
  or evidence of behavior parity.
- The user communicates business goals, unavailable business facts, product
  preferences, and exact real-world consent.
- IntentOS/Codex owns technical selection, implementation planning,
  verification, review, baseline selection, and technical risk treatment.

## Problem

The existing Active Guidance registry checks explicit files and Prompt families
but does not prove parity with:

- guidance referenced by an active Prompt, Agent, script, document, or template;
- guidance generated by `scripts/init-project.mjs`;
- platform assets listed as formal adapters;
- source files copied or rendered into an initialized project.

As a result, a registered Prompt can be current while a referenced Core file or
generated Agent appendix still delegates technical judgment to the user.

## Deliverables

### 1. Effective Guidance Graph

Extend Review Context Authority with:

- explicit active roots;
- active Prompt families;
- source generators;
- allowed reference prefixes;
- compatibility platform declarations.

The checker must recursively resolve project-local references from active roots.
Current reachable files are checked even when they were not manually listed.
Historical and compatibility files are recorded as non-active boundaries and
must not become current authority through a link.

### 2. Responsibility Semantic Close-Out

Remove or translate active instructions that ask the user to approve:

- architecture, stack, Profile, baseline, BL level, industrial packs;
- verification, review, subagent, hook, checker, or workflow mechanics;
- technical risk treatment or release readiness;
- reversible project-file or governance mechanics inside the requested scope.

Keep exact user input only for the four current decision classes.

### 3. Generator And Installed-Project Parity

`scripts/init-project.mjs` must generate the same responsibility model as the
source repository. New-project initialization and existing-project governance
migration output must be semantically scanned after generation.

### 4. Platform Boundary

- keep `platforms/codex/*` in the formal platform adapter group;
- remove Claude and Cursor from the formal adapter group;
- classify Claude and Cursor files as compatibility references;
- state that they are not installed by default and have no current parity claim.

### 5. Claim Correction

1.107.1 supersedes the over-broad 1.104.1 generated-parity claim. The corrected
claim is limited to the effective Codex guidance graph and generated Codex
assets verified by this release.

## Implementation Scope

- `core/review-context-authority.json`
- `scripts/lib/review-context-authority.mjs`
- `scripts/check-review-context-authority.mjs`
- active Core, docs, Prompt, template, checklist, and generator text reached by
  the effective guidance graph
- `platforms/claude/instructions.md`
- `platforms/cursor/rules-template.md`
- `intentos-manifest.json`
- focused tests and full self-check integration
- version, README, release, and reference records

## Non-Goals

- no implementation, apply, commit, push, release, or production authority;
- no Claude or Cursor behavior certification;
- no new public mode or technical choice for the user;
- no deletion of compatibility assets;
- no claim that semantic consistency proves business or production correctness.

## Acceptance

### Source Authority

- every effective current guidance node is project-local and non-symlinked;
- active references are recursively resolved;
- historical and compatibility references do not become current authority;
- `core/project-onboarding.md` is reached and checked from its Prompt;
- `scripts/init-project.mjs` is checked as an active guidance producer.

### Semantic Contract

- `Humans decide:` plus a technical list fails closed;
- cross-line labels and list context are checked;
- technical choice, confirmation, and approval drift fails closed;
- bounded business facts and exact real-world consent remain valid.

### Distribution

- Codex is the only formal agent adapter;
- Claude and Cursor are compatibility-only and absent from formal adapters;
- initialized Codex projects pass the same effective-guidance scan;
- existing-project Agent governance output does not reintroduce old semantics.

### Verification

- focused 1.107.1 tests pass;
- Review Context Authority check passes in source and generated projects;
- Manifest strict validation passes;
- IntentOS full self-check passes;
- `npm run verify` passes;
- `git diff --check` passes.

## Stop Conditions

Stop the release if:

- any effective current guidance node contains responsibility drift;
- a reachable current file is silently omitted;
- generated Codex assets differ from the source contract;
- Claude or Cursor remains represented as formally supported;
- a compatibility or historical file can override current guidance;
- verification relies only on source text without generated-project evidence.
