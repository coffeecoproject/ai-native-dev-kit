# Project Entry And Adoption Trust

IntentOS first works out what kind of situation it is entering, without asking
you to choose technical options.

For a new project, Codex:

1. checks that the destination is safe;
2. remembers the product goal you described;
3. prepares the exact project files internally;
4. writes them as one recoverable operation;
5. starts IntentOS from the generated project itself;
6. confirms the first task can enter the normal Work Queue and review path;
7. reports setup complete only after the receipt and checks pass.

For an existing project, Codex does not overwrite it as a template. It reads the
project, maps current work and existing rules, compares them with IntentOS, and
then chooses the safest complete adoption route. Stronger project rules remain
in force; missing or unreliable governance is repaired through a reviewed apply
plan.

You do not need to decide the framework, baseline level, migration depth,
evidence format, rollback strategy, or internal command sequence. IntentOS only
returns to you when a real business fact is missing, an external fact cannot be
obtained, a product preference is genuinely open, or a prepared real-world
action needs your consent.

Installing files is not enough. IntentOS calls itself active only after the
project can independently enter the workflow and route ordinary work without
changing files during the check.
