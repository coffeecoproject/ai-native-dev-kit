# Migration: 0.33.0 to 1.0.0

This guide describes the planned migration path from the 0.33.0 baseline to the 1.0.0 product surface.

In 0.42.0, migration is assessment-only. It can produce a plan but cannot apply changes.

## Command

```bash
node scripts/cli.mjs migrate --target ../project --from 0.33.0 --to 1.0.0 --dry-run
node scripts/cli.mjs migrate --target ../project --from 0.33.0 --to 1.0.0 --write-plan migration-plan.json
```

## Added Assets Since 0.33.0

Major added surfaces include:

- Goal Mode assets and checks
- Subagent Orchestration assets and checks
- Review Loop semantic checks
- Bounded Next-Step checks
- Engineering Baseline governance
- Output Quality and glossary checks
- manifest-authoritative asset inventory
- CLI front door
- init/update dry-run, write-plan, apply-plan, and backup flow
- artifact frontmatter and schemas
- fixture matrix expansion
- checker shared libraries
- industrial pack maturity metadata
- license boundary explanation docs
- docs information architecture and migration docs

## Removed Assets

No 0.42.0 migration command removes target project assets.

If a future migration recommends removal, it must appear in a reviewable plan with human approval.

## Renamed Assets

No automatic renames are applied by the 0.42.0 migration command.

Conceptual changes:

- long README guidance moves into operator manual and reference docs
- migration support moves from planned-only CLI placeholder to plan-only command
- machine-readable artifact fields move into frontmatter for new generated artifacts

## CI Impact

Newer projects may receive or update:

- `.github/workflows/ai-workflow-checks.yml`
- workflow artifact checks in ready or changed-only mode
- Review Loop and Bounded Next-Step checks
- Goal Mode and Subagent Orchestration checks
- platform and industrial baseline checks when selected

Existing CI should not be overwritten without reviewed plan and approval.

## AGENTS Impact

Existing `AGENTS.md` or `agent.md` may contain project-owned instructions.

The IntentOS must not overwrite those files automatically.

Use the migration report and explicit flags only after review:

- `--apply-agent-governance`
- `.intentos/migration-reports/agents-governance.md`

## PR Template Impact

Existing `.github/pull_request_template.md` may be project-owned.

The IntentOS must not overwrite it automatically.

Use explicit review and apply only when accepted:

- `--apply-pr-template-governance`
- `.intentos/migration-reports/pr-template-governance.md`

## Human Approval Requirements

Human approval is required for:

- adopting IntentOS into governed or production projects
- overwriting or merging agent instructions
- overwriting or merging PR templates
- enabling BL2 industrial packs
- accepting migration reports
- changing CI behavior
- treating legacy artifacts without frontmatter as ready
- accepting license wording risk before 1.0 release materials
- any migration apply behavior in a future phase

## Rollback

For plan-only migration, rollback is simply deleting the generated plan file.

For reviewed init/update apply-plan, rollback depends on the project:

- use `--backup-dir <dir>` when overwrites are possible
- revert only workflow asset changes
- do not revert user business code
- preserve migration reports and final reports as evidence unless the project owner asks otherwise

## Stop Conditions

Stop if:

- the project is production or dirty and no adapter decision exists
- migration would require direct target writes
- existing governance ownership is unclear
- CI impact cannot be explained
- AGENTS or PR template merge requires human decision
