# 1.19.0 Baseline Selection Precision Calibration Plan

## Purpose

1.19.0 turns baseline selection calibration from narrative notes into
checkable precision evidence.

The goal is not to add more baseline packs. The goal is to make Codex better at
answering:

```text
What is safe now?
What is only a candidate later?
Which platforms are active, deferred, possible, or not detected?
Where did the selector over-recommend or miss a signal?
```

## Inputs

1.19.0 builds on:

- `baseline-calibration-reports/scoreboard.md`
- `scripts/resolve-guided-baseline-selection.mjs`
- `scripts/check-guided-baseline-selection.mjs`
- sanitized read-only 1.18 reports
- 1.19 synthetic precision fixtures generated by
  `scripts/check-baseline-selection-precision.mjs`

## Scope

1.19.0 adds:

- `scripts/check-baseline-selection-precision.mjs`
- machine-checkable scoreboard rules
- synthetic precision fixture cases
- a 1.19 calibration report
- release evidence and version metadata

## Required Behavior

### Scoreboard Validation

The precision checker must validate:

- required scoreboard columns
- unique kebab-case case ids
- false positive values: `yes`, `no`, `monitor`
- false negative values: `yes`, `no`, `monitor`
- fix status values: `fixed`, `pending`, `monitor`, `not-applicable`
- explicit boundary language that the scoreboard is not production validation

### Synthetic Precision Fixtures

The checker must run local synthetic fixtures for:

- Mini Program cloud functions
- permission-only docs
- active Web admin
- production governed read-only project
- dirty worktree with payment risk
- monorepo with deferred platforms
- backend data/API project
- empty unknown new project

Each case must compare resolver output against expected state, safe action,
platform states, standard packs, and BL2 candidate language.

### Human Boundary

The checker can prove that the selector matched expected synthetic cases. It
must not claim:

- real production validation
- commercial readiness
- target-project write approval
- implementation approval
- release/production approval
- security/privacy/compliance/payment/migration approval

## Non-Goals

- Do not add new standard packs.
- Do not add new industrial packs.
- Do not promote draft packs.
- Do not make BL2 default.
- Do not enable industrial packs automatically.
- Do not make calibration fixtures a substitute for real project review.
- Do not write to real target projects.

## Verification

Required checks:

```bash
node scripts/check-baseline-selection-precision.mjs .
node scripts/check-baseline-selection-precision.mjs . --skip-fixtures
npm run verify:baseline
npm run verify:release
npm run verify
```

## Release Boundary

1.19.0 is a calibration release. It improves evidence quality and precision
tracking for baseline recommendations. It does not change target-project write
authority or baseline-pack activation authority.
